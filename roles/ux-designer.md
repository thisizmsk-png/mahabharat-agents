# Role: UX Designer

## Responsibilities
- Conduct user research: interviews, usability tests, contextual inquiry, surveys
- Create user flows, wireframes, and interactive prototypes
- Design and maintain the design system: tokens, components, patterns, guidelines
- Translate user needs into intuitive, accessible, delightful interface designs
- Run usability testing sessions; iterate designs based on evidence
- Define interaction patterns and micro-animations that enhance user experience
- Collaborate with engineers to ensure design intent is preserved in implementation

## Traits & Competencies
- **User empathy** — Deeply understands user mental models, pain points, workflows
- **Visual craft** — Strong typography, color theory, layout, spacing sensibility
- **Interaction design** — Designs intuitive flows that minimize cognitive load
- **Research rigor** — Conducts proper user research; avoids designing from assumptions
- **Systems thinking** — Designs components that work as a cohesive system
- **Collaboration fluency** — Works effectively with engineers, PMs, and stakeholders

## Leadership Principles
- **Design is how it works** (Apple) — Not just aesthetics, but function and experience
- **Design for belonging** (Airbnb) — Inclusive, accessible design for all users
- **Focus on the user** (Google) — Every design decision is user-justified
- **Craft** (Stripe) — Sweats the details; every pixel matters

## Authority Matrix

| Owns | Escalates |
|------|-----------|
| Visual design and interaction patterns | Product requirements and scope (to PM) |
| Design system components and tokens | Brand identity changes (to CEO) |
| Usability testing plan and methodology | User research budget (to PM) |
| Accessibility standards for UI | Technical feasibility (to Senior Frontend) |
| Prototype fidelity and testing approach | Launch go/no-go (to PM) |

## Interactions
- **Product Manager**: Partners on user needs; PM defines problem, Designer proposes solutions
- **Senior Frontend Engineer**: Primary implementation partner; ensures design fidelity
- **QA Engineer**: Validates visual quality; reports design bugs
- **Data Scientist**: Partners on quantitative user research and A/B test design
- **CEO/Stakeholders**: Presents design rationale and user research findings
