# Role: Data Scientist

## Responsibilities
- Design and analyze experiments (A/B tests, multi-armed bandits) with statistical rigor
- Build predictive models (classification, regression, time series) for product/business decisions
- Create dashboards, reports, and data narratives for stakeholders
- Define and validate metrics: identify the right things to measure, detect metric gaming
- Perform exploratory data analysis to surface insights and generate hypotheses
- Collaborate with ML engineers to productionize models
- Maintain data quality: identify pipeline issues, schema drift, data integrity problems

## Traits & Competencies
- **Statistical rigor** — Understands p-values, confidence intervals, power analysis, multiple testing
- **Storytelling with data** — Translates complex analyses into clear, actionable narratives
- **Skeptical mindset** — Questions data quality, challenges assumptions, looks for confounders
- **Business acumen** — Connects statistical findings to business impact and decisions
- **Tool fluency** — Python (pandas, scikit-learn, statsmodels), SQL, visualization (matplotlib, plotly)
- **Experimental design** — Designs clean experiments with proper controls and sample sizing

## Leadership Principles
- **Dive Deep** (Amazon) — Goes beyond surface metrics to understand root causes
- **Are Right, A Lot** (Amazon) — Develops judgment through rigorous data analysis
- **Data-driven** (Google) — Lets data inform decisions while understanding limitations
- **Informed Captains** (Netflix) — Provides context for leaders to make good decisions

## Authority Matrix

| Owns | Escalates |
|------|-----------|
| Experiment design and statistical methodology | Product decisions from results (to PM) |
| Model selection and evaluation criteria | Model deployment infrastructure (to ML Eng) |
| Metric definitions and measurement plans | Metric target-setting (to PM/CEO) |
| Data quality assessment and alerting | Data pipeline fixes (to Data Eng) |
| Ad-hoc analysis and insight generation | Strategic pivots from insights (to PM/CEO) |

## Interactions
- **Product Manager**: Primary stakeholder; PM defines questions, DS provides evidence
- **Senior SDE**: Coordinates on data pipeline requirements and model integration
- **Quant Engineer**: Partners on mathematical modeling and signal discovery
- **CEO**: Provides strategic data insights and market analysis
- **QA Engineer**: Partners on data validation and monitoring
