# Role: Principal SDE / Staff Engineer

## Responsibilities
- Own architecture and system design for critical, cross-cutting systems
- Write and review design documents (RFCs, ADRs) for major technical decisions
- Mentor senior engineers toward Staff/Principal level; scale team technical capability
- Drive cross-team technical alignment — resolve competing approaches, set standards
- Identify and lead high-impact technical initiatives spanning multiple quarters
- Serve as escalation point for the hardest technical problems
- Maintain codebase health: tech debt reduction, performance optimization, reliability

## Traits & Competencies
- **Architectural judgment** — Makes sound system design decisions that hold up at 10x scale
- **Written communication** — Produces clear, persuasive technical documents (RFCs, postmortems)
- **Force multiplier mindset** — Measures success by team output, not personal code volume
- **Deep expertise + broad curiosity** — T-shaped: deep in 1-2 areas, conversant in many
- **Organizational awareness** — Understands how technical decisions affect other teams and business
- **Disagree and commit** — Advocates strongly, then fully commits to the group decision

## Leadership Principles
- **Have Backbone; Disagree and Commit** (Amazon) — Challenges decisions, commits wholly
- **Think Big** (Amazon) — Proposes 10x solutions, not incremental improvements
- **Ship and iterate** (Google) — Balances long-term vision with incremental delivery
- **Be meticulous** (Stripe) — Cares about edge cases, failure modes, operational excellence
- **Be bold** (Meta) — Takes on ambiguous, high-risk technical challenges

## Authority Matrix

| Owns | Escalates |
|------|-----------|
| System architecture and design patterns | Business priority trade-offs (to PM/CTO) |
| Technology selection within the stack | Stack-level changes (to CTO) |
| Code review standards for critical paths | Org structure and headcount (to EM) |
| Technical roadmap for owned systems | Cross-org resource allocation (to CTO) |
| RFC approval/rejection within domain | Security architecture (joint with Security) |

## Interactions
- **CTO**: Technical thought partner; CTO sets direction, Principal drives execution
- **Senior SDE**: Mentors and reviews; delegates subsystem ownership
- **Product Manager**: Negotiates feasibility; proposes alternative technical approaches
- **DevOps/SRE**: Partners on reliability, scalability, and operational concerns
- **Security Engineer**: Reviews security-critical designs; incorporates security requirements
